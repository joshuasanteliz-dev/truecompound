# Psychological UX Audit

Review the page as a user experience, not just an interface.

Analyze:

1. Attention
- Where does the eye go first?
- Is that where attention should go?
- Is there visual noise competing with the primary goal?

2. Motivation
- Why would the user continue scrolling or interacting?
- Is the value proposition emotionally and practically clear?

3. Trust
- What reduces skepticism?
- Are there proof points, specificity, transparency, or credibility signals?
- Does anything feel fake, exaggerated, generic, or AI-written?

4. Cognitive load
- Is the page easy to understand?
- Are sections chunked logically?
- Are choices too many, too vague, or poorly ordered?

5. Emotion
- What should the user feel?
- Does the design create that feeling through typography, color, spacing, copy, imagery, and interaction?

6. Friction
- What could stop the user?
- Are CTAs clear?
- Are forms, links, navigation, and next steps obvious?

7. Memory
- What makes the page memorable?
- Is there a distinctive visual or conceptual hook?

8. Conversion
- Is the desired action clear?
- Are nudges ethical and non-manipulative?
- Does the page respect user autonomy?

Output:
- Critical UX psychology issues
- Missed user needs
- Trust gaps
- Cognitive load problems
- Emotional mismatch
- Friction points
- Highest-impact improvements
- What should remain restrained
