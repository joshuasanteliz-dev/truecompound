# Motion Interaction Review

Motion must serve the experience. It is not decoration.

Before adding motion, define:
- What does this motion communicate?
- Does it guide attention?
- Does it clarify hierarchy?
- Does it make the interface feel alive without becoming distracting?
- Does it support the page's emotional direction?
- Is it appropriate for the audience?
- Does it work on mobile?
- Does it respect reduced-motion preferences?
- Is it performant?

Avoid:
- generic fade-up-on-scroll everywhere
- excessive parallax
- cursor gimmicks that reduce usability
- animation that hides content
- animation that delays the user's task
- motion that creates layout shift
- motion that harms readability

Prefer:
- restrained transitions
- meaningful hover/focus states
- subtle section reveals
- purposeful hero motion
- state changes that explain interaction
- CSS transforms/opacity over layout-heavy animation
- reduced-motion fallbacks

Output:
- where motion helps
- where motion should be avoided
- recommended interaction per section
- performance risks
- accessibility requirements
